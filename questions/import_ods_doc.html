Format du fichier d'import (.ods)

← Retour à l'import
Format : tableur .ods  ·  Onglet : Résultats de la requête  ·  1 ligne = 1 question  ·  40 colonnes organisées en 5 blocs

Bloc 1 — Identité de la question

ColonneTypeObligatoireDescription
IDentier✅ ouiIdentifiant unique de la question. Si l'ID existe en base → mise à jour ; sinon → création.
Texttexte long✅ ouiÉnoncé complet de la question.
HinttextenonIndice optionnel affiché avant la réponse.
Typeenum✅ ouiQCM · QCM-RM · VF
Difficultyentier 0–4✅ oui0 = très facile · 4 = expert
Languageenum✅ ouiFRENCH · ENGLISH · ITALIAN · GERMAN

Bloc 2 — Réponses

ColonneTypeObligatoireDescription
Answer Choice Atexte✅ ouiProposition A
Answer Choice Btexte✅ ouiProposition B
Answer Choice Ctextesauf VFProposition C
Answer Choice DtextenonProposition D (absente pour les VF)
Answer Correctenum✅ ouiLettre(s) de la/des bonne(s) réponse(s) : a b c d — ou combinaison : ab, acd…
Has Ordered Answersbooléen✅ ouiTrue si l'ordre des propositions est figé, False si aléatoire. Toujours True pour les VF.
Answer Explanationtexte long✅ ouiExplication pédagogique affichée après la réponse.

Pour un VF, ne remplir que Answer Choice A et Answer Choice B. Answer Correct vaut a ou b.

Règles de formatage importantes

  1. Tag List : toujours encadrés d'accolades {}. Les tags contenant des espaces doivent être entre guillemets doubles : {"Question Choc","Culture générale",Biodiversité}
  2. Quiz List : même format : {1,78,116}
  3. Answer Correct : toujours en minuscule (a, b, c, d)
  4. Has Ordered Answers : écrire True ou False (majuscule initiale)
  5. Colonnes vides : laisser vides — ne pas mettre NaN, null ou –
  6. Dates : format ISO YYYY-MM-DD pour Created et Validation Date, YYYY-MM-DD HH:MM:SS pour Updated
